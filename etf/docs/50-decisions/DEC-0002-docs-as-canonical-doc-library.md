# DEC-0002 使用 docs/ 作为统一文档主库

状态：Accepted
日期：2026-04-15

## 背景

项目已有多份历史资料，但来源分散、时间跨度不同、口径不完全一致。若继续在旧文件上追加，后续会越来越难判断哪一页代表当前真相。

## 决策

使用 `/opt/data/scripts/etf/docs/` 作为当前项目的统一文档主库。

执行原则：
1. 新结论优先写入 docs/
2. 旧资料先索引，不强制迁移
3. 正式结论以 docs/ 与 DEC 记录为准

## 原因

1. 保持工作区整洁，不打乱旧文件位置
2. 让当前执行文档、研究文档、决策文档有统一入口
3. 后续便于多智能体或未来自己接手

## 证据

- `../README.md`
- `../80-archive/legacy-notes-index.md`
- `../00-index/current-status.md`

## 影响范围

- 项目导航
- 阶段计划
- 结果汇总
- 决策追溯
- 历史资料管理

## 未采纳方案

### 方案 A：直接在旧文档上持续修改
未采纳原因：旧资料混有历史上下文，直接追加容易污染当前执行口径。

### 方案 B：立即迁移并重命名所有旧文档
未采纳原因：当前不需要大规模整理旧文件，容易打乱已有工作区。

## 后续动作

1. 所有新实验与结论统一进入 docs/
2. 历史资料继续只做索引，不做批量搬迁
