## 5. trigger_deviation 差异分析

### trigger = 0.15
- 全样本仅有 13.69% / 0.90，在三组中最弱。
- 测试期为 19.88% / 1.45，明显落后于 0.20 与 0.25。
- Walk-Forward 只有 0.56，说明触发阈值过低后，策略更容易响应噪音而不是高质量信号。

### trigger = 0.20
- 作为此前默认值，整体表现仍然稳健：全样本 13.83% / 0.92，测试期 23.64% / 1.70。
- 相比 0.15 明显更好，说明提高触发阈值确实能过滤一部分低质量调仓。
- 但在 full sample、test、walk-forward 三层上都被 0.25 超越，因此更适合作为次优对照，而不是最终主方案。

### trigger = 0.25
- 全样本最佳：14.47% / 0.99。
- 测试期最佳：26.67% / 2.02 / -8.45%。
- Walk-Forward 最佳：13.95% / 1.01 / -8.04%。
- 调仓次数最少，仅 112 次，说明更高阈值不仅没有压制收益，反而更有效地过滤了无效波动。

## 6. 关键发现

1. **trigger 过低会带来噪音响应**：0.15 在三层验证中全部落后，说明过于灵敏的触发机制会把更多非必要变动当成信号。
2. **0.20 是稳健值，但不再是最优值**：它仍然合格，但已经被 0.25 在长期、样本外和滚动验证中全面超越。
3. **0.25 呈现“全面胜出”而非边缘领先**：full sample、test、walk-forward 全部领先，同时调仓次数更少，说明主线可以明确升级，而不是只做观察保留。
4. **主线在第五阶段出现确定性增强**：这不是“勉强维持旧参数”，而是新的更优阈值被清楚识别出来。
